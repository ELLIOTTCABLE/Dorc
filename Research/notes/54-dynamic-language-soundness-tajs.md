# Sound AI of a messy dynamic language — TAJS lessons (our exact predicament)

TAJS (Jensen, Møller, Thiemann) is the closest methodological precedent: a **sound** static analysis of JavaScript — weak/dynamic typing, silent coercions, dynamic property names, prototype chains, `eval`, `with`, variadic functions. Shell is the same *kind* of beast (dynamic, coercion-happy, recalcitrant). Same author/group as the SPA textbook and `shstats` — the Aarhus throughline.

## Directly transferable design decisions
- **Sound AI via the monotone framework over a deliberately-intricate lattice.** Soundness is the *stated requirement* (so absence-of-warning = proof). This is the **over-approximating-AI flavor** Dorc wants (note 51's soundness-bias) — *not* Infer's unsound bug-finding. TAJS shows it's feasible and useful on the small/medium programs that dominate ("targeted at hand-written programs of a few thousand LOC; most existing JS programs fit") — which *matches the shell corpus evidence* (provisioning scripts are short). Analysis time seconds–minutes; precision good on small/medium, degrades on the largest.
- **`eval`/dynamic code construction → the analyzer cannot model it → emit a warning and treat as the boundary** ("the fixpoint solver requires the node/label sets to be fixed"). They note this is "likely satisfactory as these functions are mostly used in stylized ways." → This is *exactly* Dorc's `unsafe`/⊤ boundary for shell `eval`/dynamic-command/`source "$dyn"`, and the corpus data backs the "rare + stylized" claim (eval ~9% / 4%). Validation: collapse dynamic-code-construction to ⊤+warning; don't try to model it.
- **Recency abstraction = the key precision lever.** Each allocation site is split into a **singleton** (`@ℓ`, the most-recently-created entity — supports **strong updates**, precise) and a **summary** (`ℓ`, older/unknown entities — **weak updates**). Disabling it craters precision (87%→<2% on one benchmark). → **Dorc analog (important):** distinguish *the specific state-entity a command just established/touched* (strong update — "package `foo` is now exactly present at version X") from *summarized/unknown entities* (weak update). Without this, the mutation domain loses precision the same way — "maybe-present" proliferates and nothing is skippable. **Adopt a recency-style strong/weak split in the effect/fact domain.**
- **Flow-graph IR of spec-level instructions** (`read/write/delete-property`, `read/write-variable`, `call/construct/after-call`, `if`, `entry/exit`…), close to the language spec; **call/return edges added on-the-fly during fixpoint** (higher-order ⇒ call graph discovered during analysis). → Dorc: command effects are spec-level ops on system-state (`pkg-install`, `file-write`, `svc-enable`); for shell, "touch the filesystem/package-db/service" are the property-read/write analogs. Discover the call graph (functions/`source`) on the fly.
- **`Modified` component** tracks what's changed since function entry, to **discard infeasible cross-call flow** (precision). → maps to per-role "what state did this role touch" — both a precision device and the establish-fact set.
- **Context-sensitivity matters** (heuristic: separate abstract states by `this`); finite-height not required — termination from program structure (finite property names drawn only from write instructions). → Dorc's fact-domain is finite (bounded by the script's literal commands/paths), so termination is unproblematic; context-sensitivity (per-host? per-role-invocation?) is a precision dial.

## Net for synthesis
TAJS is the proof that **sound, precise abstract interpretation of a dynamic, messy language is achievable for the small/medium programs that dominate** — with the *unanalyzable dynamic core* (`eval`) handled by ⊤+warning and **recency/strong-weak abstraction** as the precision keystone. Dorc's Q1 analysis is the same shape: monotone-framework AI over a custom (system-state) lattice, over-approximating, ⊤-on-dynamic, recency-style strong/weak updates on state-entities. The corpus evidence (short, eval-rare scripts) says this lands in TAJS's sweet spot, not its hard cases.
