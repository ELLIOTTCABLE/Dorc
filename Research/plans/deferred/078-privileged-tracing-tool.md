# Deferred: the privileged tracing tool (vague shape, very deferred)

**Status: deferred devtooling, NOT core.** This is post-"language exists." It is a **separate command + codebase**, published alongside, **installed separately** (the user's clinical-quarantine requirement — it's privileged, so it must never be linked into the agentless core). The core tool's only obligation to it *now* is the hook surface (`077-pluggability-and-hook-surface.md`). This doc is a sketch to aim that surface correctly, not a plan to build.

## Purpose (the go/no-go that justifies it existing at all)
Bootstrap oracle authoring for **complex, owns-the-world ops tooling** (docker, k8s, terraform-wrappers, vendor installers) **without Dorc knowing anything tool-specific.** The user's hard go/no-go: *if it can't derive useful dependency info from `docker compose up foo.yaml` knowing nothing about docker, it isn't worth building.* We will never hand-implement per-tool support for the whole heterogeneous ops universe; a tool-agnostic tracer is the only way to cover it. Input: a Dorc oracle/script + a fixture or sacrificial host. Output: **proposed `sh` guards** (`if [ -f /mnt/blah ]; then …`) + flagged hidden deps + cost-class hints. **No metadata — the deliverable is a patch to a shell oracle.** One author runs it once; the resulting guards benefit the whole community via the oracle library.

## The mechanism finding that shapes everything: per-process tracing structurally fails here
Modern ops tooling is **daemon-mediated**: `docker compose up` is a thin (Go, static) client that talks to `dockerd` over a unix socket; `dockerd → containerd → runc` do the real filesystem/network/mount work — **in process trees that are NOT children of the traced command.** The bind-mount of `/mnt/blah` is a *daemon* syscall, not the CLI's. Consequence (confirmed verbatim by rattle's own README: *"if `fsatrace` runs a binary that goes through a server process not spawned by the build system, it won't be visible"*):
- **LD_PRELOAD** (fsatrace/rattle): defeated **twice** — Go static-links (no libc to wrap) *and* the work is out-of-process. Useless for exactly the tools that matter. → **the user's instinct is right: LD_PRELOAD is a distraction here**, at most a near-free defense-in-depth catch for simple dynamically-linked leaves.
- **ptrace** (strace/bigbro): sees static binaries (syscall boundary) and follows *children* — but `dockerd` is **not a child**, it's reached via socket. To trace it you'd attach to dockerd specifically = **tool-specific knowledge**, which fails the go/no-go. → insufficient alone.
- **system-wide observation (eBPF, or auditd on older kernels): the only mechanism that passes the go/no-go.** It observes *all* processes' syscalls in a time window regardless of process topology, so it catches dockerd/containerd/runc/the-container doing their thing — **without knowing they're docker.** This is the real value, and it's the hard/privileged one. The user's read ("the value is in the harder approaches") is correct.

## The honest hard problem: attribution
System-wide tracing sees *everything the machine did* during the window, including unrelated background noise (cron, other daemons). Turning that into "what did `docker compose up` cause" is heuristic:
- **Baseline-diff** (what does the idle host do in an equal window) + **time-window scoping** + **new-process-subtree capture** (eBPF can watch process creation system-wide, so containerd→runc→container-proc spawned *during* the window are attributable even though they're not the CLI's children) + optional **cgroup correlation** (precise but edges toward tool-specific).
- → tool-agnostic but **noisy**; output is *candidate* guards for human review, not ground truth. That's fine — it matches the whole "advisory, honest, human-adjudicated" posture. State the noise; don't pretend precision.

## The easy/hard security split (the user's synthesis, made concrete)
- **EASY-to-secure → lives in the CORE tool, all users** (`077-pluggability-and-hook-surface.md`): **seccomp-BPF** classification of `socket(AF_INET/INET6)`. Unprivileged, fail-closed, mainline. Gives network-cost-class + undeclared-network backstop for the probe's *own* process tree, at probe- AND apply-time, for everyone — the "wide community backstop." Coarse; blind to daemons. **NOT expected to catch most things** — it's the cheap wide net.
- **HARD-to-secure → this deferred privileged tool**: eBPF/auditd system-wide tracing for rich fs+net deps incl. daemon-mediated. Needs root/`CAP_BPF`. **Quarantined** in its own installable binary.
- **DISTRACTION**: LD_PRELOAD (above). Maybe a free extra route; not the focus.
- **"Is an easy-to-secure LD_PRELOAD possible?" (the user's noted-not-dived question):** SUSPECT no, and it's moot — LD_PRELOAD's problem isn't security, it's *coverage* (static + daemon blindness). The easy-to-secure niche is already filled better by seccomp, which is fail-closed in a way LD_PRELOAD (arbitrary code injection) never is. So: don't chase a secure LD_PRELOAD; use seccomp for easy, eBPF for hard.

## Complement (orthogonal, also deferred): containerized fixtures — for reproducibility, NOT security
A separate tooling dimension (the user's): run oracle tests in throwaway containers for **reproducibility/CI**, making **no security claims** (sandboxing-with-security-guarantees is a different *order* of hard problem — don't claim it, can't perfectly deliver it; "security-fail-fast" instead: *"trace this in an environment you're willing to sacrifice."*). It **complements** tracing: in a disposable container you can **lift further** — actually *run* the destructive thing and observe it — catching the highest-value cases that neither probe-time nor apply-time would surface. Canonical example: `foo --dry-run` that, when a core component is missing, "helpfully" triggers a slow real package-install. That landmine is exactly what you want to detect once, in a sacrificial container, and annotate as a guard in the oracle. Tracing + containerization are orthogonal-but-concurrent; both feed the same output (sh guards).

## Integration sketch (how it plugs in, when it eventually exists)
Consumes the hook surface: runs Dorc in **`--faithful` mode** (one-leaf-one-exec, attributable) under system-wide tracing + optionally inside a fixture container; correlates syscalls → leaf-IDs → source AST nodes (via the dual-use provenance map); emits **candidate `sh` guard patches** + cost-class annotations to the oracle source for human review. Both probe and apply runners are wrappable, so it can trace either phase. Nothing about it reaches into Dorc internals — it wraps Dorc the way it wraps any traced program.
